# NoteBookLM 使用技巧

- 先搭技能体系框架，再用 NotebookLM 填充框架中的知识技能
- 当不知道框架节点内部结构时，可以先用概念问题让 NotebookLM 反向生成子框架

## 整体工作流
> 自然提问-->结构化追问-->实践追问-->沉淀笔记

- 工作流：
```text
  先搭一级技能体系
    ↓
  每次选一个知识点
    ↓
  先用自然语言提问：请详细讲解 xxx
    ↓
  再追问：请拆成学习框架
    ↓
  再追问：请设计实践练习
    ↓
  最后沉淀成自己的笔记和项目代码
```

- 示例：
```text
  RAG技能体系
    ↓
  Chunk
    ↓
  请详细讲解 RAG chunk
    ↓
  请把 RAG chunk 拆成学习框架
    ↓
  请设计 RAG chunk 的 Python 实践练习
    ↓
  整理成自己的 chunk 学习笔记
```

## 分类拆解

#### 1> 概念讲解型
适合你刚知道一个名词。

- 模板：
```text
  请详细讲解【概念】。
  要求：
  1. 用通俗语言解释
  2. 说明它解决什么问题
  3. 说明它在系统中的位置
  4. 怎么使用，举一个实际项目例子
```

- 示例：
```text
  请详细讲解 RAG chunk。
```

#### 2> 知识框架型
适合你已经知道概念大概意思，想系统化。

- 模板：
```text
  请把【概念】拆成一个学习框架。
  要求：
  1. 拆出核心子概念
  2. 说明每个子概念的作用
  3. 按初级、进阶、工程化分层
  4. 给出学习顺序
```
- 示例：
```text
  请把 RAG chunk 拆成一个学习框架。
```
#### 3> 对比辨析型
适合你遇到多个相近概念。

- 模板：
```text
  请对比【A】和【B】。
  要求：
  1. 分别解释
  2. 说明核心区别
  3. 说明使用场景
  4. 给出工程实践建议
```
- 示例：
```text
  请对比 chunk size 和 chunk overlap。
```

#### 4> 实践落地型

- 模板
```text
  请设计一个【概念】的实践练习。
  要求：
  1. 从最小可运行 demo 开始
  2. 逐步增加复杂度
  3. 给出 Python 代码
  4. 说明每一步练习目标
  5. 说明常见错误
```
- 示例：
```text
  请设计一个 RAG chunk 的 Python 实践练习。
```
